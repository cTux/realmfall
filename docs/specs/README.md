# Specs

`docs/specs` stores the project's implementation-facing specifications.

Reference indexes:

- [Gameplay Features](./reference/gameplay-features/README.md)
- [Technical Solutions](./reference/technical-solutions/README.md)

Workflow expectations:

- Every implemented feature should create or update the matching spec.
- Every change to an existing feature should keep its spec in sync in the same task.
- Each gameplay feature and each technical solution should have its own dedicated spec file.
- Use index documents as navigation only.
